# 狗哥配图

把文章中的关键判断、认知转折、流程、状态和隐喻，转成可插入正文的 16:9 写实 3D 极简棚拍场景。主角 **AI狗哥** 参与场景互动，让抽象概念变得可感可亲。先解释"哪里值得画、画什么"，再按用户明确指令生成图片。

## 按需读取参考

- 规划视觉风格或判断材质、光影与文字时，读取 `references/visual-dna.md`。
- 选择场景结构、发明物理隐喻或判断是否需要 AI狗哥时，读取 `references/composition-patterns.md`。
- 准备生成或编辑图片时，读取 `references/prompt-template.md`。
- 检查生成结果或决定是否重生成时，读取 `references/qa-checklist.md`。
- 需要精确还原 AI狗哥形象时，读取 `references/character-sheet-ai-dog-bro.md` 并查看 `assets/character-turnaround-ai-dog-bro.jpg`。

不要创建或使用 SVG、HTML/CSS、Canvas、PIL、本地脚本或任何"仿 3D"占位图。真正生图时只能使用内置 `image_gen`，不要索取 API key。

## 工作流

### 1. 读取内容

读取用户提供的正文、Markdown、文件、截图或链接。链接可访问时先读取页面正文；无法访问时说明限制并请用户粘贴正文，不要仅凭标题编造文章内容。

提炼：
- 全文核心判断与主要受众。
- 承担认知转折的段落。
- 值得视觉化的流程、结构、状态、对比或隐喻。
- 只适合文字、不值得配图的段落。

不要平均分配配图。优先选择能帮助理解、形成记忆或缓解阅读密度的认知锚点。

### 2. 先给配图策略

除非用户已经明确要求生成图片，否则只输出策略，不调用 `image_gen`。

短文推荐 1–3 张，常规或长文推荐 3–6 张。除非用户明确要求，不要超过 6 张。按阅读价值排序，并为每张图写清：

1. 插入位置。
2. 认知锚点。
3. 核心意思。
4. 场景隐喻。
5. 构图与镜头。
6. 主要物件与材质。
7. 是否需要 AI狗哥，以及 AI狗哥需要承担的核心动作。
8. 1–5 个可选短中文标注。
9. 生成优先级。

每张图只解释一个核心意思。场景必须源自当前文章，不复刻既有案例或模板构图。

### 3. 判断是否生成

把"分析怎么配图、给方案、给 shot list、建议插图"视为策略请求，不生成图片。

只有用户明确说"生成图片、生成配图、做图、出图、输出 PNG"等操作性指令时才生成。用户已明确要求生成时，先形成简短策略，然后直接生成，不再要求确认。

- 用户指定数量时，按指定数量生成。
- 用户未指定数量时，生成策略中的全部入选图。
- 用户只指定一个概念时，只生成一张。

### 4. 逐张并发生成

生成前读取 `references/prompt-template.md` 和 `references/character-sheet-ai-dog-bro.md`。为每个入选场景准备独立提示词，每次 `image_gen` 调用只生成一张完整的 16:9 图片，绝不把多张配图拼成宫格或合集。

有多张图且运行环境支持并发时，同时发起独立调用；不支持并发时再逐张生成。某一张失败时只重试该张，不要重复已经成功的图片。所有图片保持同一视觉 DNA，但场景、物件和构图分别服务各自认知锚点。

必须生成真实位图。禁止用 SVG、HTML/CSS、Canvas、PIL、脚本、流程图组件、3D 软件截图或预制占位图替代 `image_gen` 输出。

### 5. 检查与迭代

生成后读取 `references/qa-checklist.md`，逐张检查画幅、白底、3D 材质、空间接触、光影、留白、文字、角色必要性和语义准确性。

存在明显失败项时优先重生成或针对问题编辑：
- 像扁平插画、PPT、海报或正式流程图。
- 材质像廉价塑料或没有厚度。
- 光源、阴影、反射或物件接触关系不成立。
- AI狗哥只是装饰，没有参与核心动作。
- 中文过多、错字或不可读。
- 一张图塞入多个观点。

编辑用户提供的图片时，只修改指定问题，保留其余构图、材质、灯光、画幅和清晰度。

### 6. 保存交付

在 workspace 中工作时，将最终图片保存到：

```text
assets/<article-slug>-illustrations/
01-topic.png
02-topic.png
```

使用简短英文 slug 和主题名。保存前检查同名文件；如已存在，追加递增编号，不覆盖现有资产。若图像工具只返回托管结果而没有本地文件，直接返回该生成结果，不要用其他绘图方式伪造本地位图。

遵守内置图像工具的交付规则，不在最后一次生成后追加与图片无关的说明。
